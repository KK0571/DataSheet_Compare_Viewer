# Chip DataSheet Compare Viewer
 (芯片规格书比对工具)

## 1. 项目背景与初衷
在硬件工程开发中，对比两款相似芯片（如 Pin-to-Pin 替代料或同类竞争型号）的参数是高频且枯燥的任务。
* **痛点**：传统的 PDF 阅读器无法有效实现双文件"同步滚动"，导致在对比 Electrical Characteristics 表格时需要频繁切换窗口或手动对齐。
* **目标**：构建一个基于 Web 的轻量化工具，利用 **第一性原理** 解决双 PDF 的同步视图与操作对齐问题。

---

## 2. 使用说明

### 2.1 基本操作

| 操作 | 说明 |
|------|------|
| 拖入 PDF | 将芯片规格书拖入左右窗口 |
| 滚轮滚动 | 两个视图同步滚动 |
| Shift + 滚轮 | 只滚动当前鼠标所在视图 |
| 拖动滚动条 | 单独滚动，不与另一边联动 |
| Ctrl + 滚轮 | 缩放当前视图 |

### 2.2 搜索功能

| 功能 | 说明 |
|------|------|
| 搜索框 | 顶部工具栏输入关键词，回车或点击搜索按钮 |
| 过滤选项 | 点击"过滤"下拉菜单，可多选：区分大小写、全字匹配、正则表达式 |
| 搜索结果 | 点击侧边栏箭头展开，点击结果跳转到对应页面 |
| 高亮显示 | 匹配的文本在 PDF 中高亮显示（黄色背景） |

### 2.3 侧边栏

| 功能 | 说明 |
|------|------|
| 默认状态 | 页面载入时两侧边栏默认收起 |
| 展开/收起 | 点击边栏顶部的箭头按钮切换 |
| 搜索结果列表 | 显示左/右侧 PDF 的搜索结果数量和摘要 |

## 3. 技术实现架构 (Implementation)

### 3.1 核心栈
* **渲染引擎**：[PDF.js](https://mozilla.github.io/pdf.js/) (v3.11.174) - 解决原生浏览器 `<embed>` 标签无法受脚本精细控制的问题。
* **交互逻辑**：原生 JavaScript (Vanilla JS) + CSS Flexbox。

### 3.2 滚动同步机制

#### 滚轮同步
- 监听 `wheel` 事件，将滚动距离同步应用到两个面板
- 按住 Shift 键时触发单独滚动模式

#### 滚动条解耦
- 通过检测鼠标按下位置是否在滚动条区域来判断用户是否在拖动滚动条
- 拖动滚动条时，另一侧不会跟随滚动

### 3.3 缩放机制
- 使用 CSS `transform: scale()` 实现快速缩放，无需重新渲染
- 缩放范围：50% ~ 300%
- 状态栏实时显示当前缩放比例

### 3.4 搜索机制
- 使用 PDF.js 提取文本内容实现搜索
- 支持正则表达式搜索
- 匹配结果添加单词边界限制，避免部分匹配

## 4. 后期维护与升级路线 (Roadmap)

### 4.1 视图优化 (UI/UX)
- [x] **同步缩放 (Sync Zoom)**：实现双窗格联动放大/缩小。
- [x] **单侧解耦**：按住 Shift 键或拖动滚动条时可以单独查看不同页码。
- [x] **搜索功能**：支持文本搜索、高亮显示、结果列表。

### 4.2 硬件工程师专属功能 (Domain Specific)
- [ ] **AI 参数提取**：集成 LLM API，自动提取两份文档中的 `V_in`, `I_q`, `R_ds(on)` 等参数并生成对比表格。
- [ ] **视觉差异高亮**：通过 Canvas 像素对比，尝试识别表格中数值不同的区域（Diff Overlay）。
- [ ] **本地存储**：利用 `indexedDB` 缓存最近对比过的 PDF，避免刷新页面后重复拖入。

## 5. 部署与使用

### 5.1 离线使用
本工具已集成 PDF.js 库，无需网络连接即可运行。

```
chip_diff_viewer/
├── Datasheet_Comparator.html  # 主程序
├── readme.md                  # 说明文档
└── js/
    ├── pdf.min.js           # PDF.js 主库
    └── pdf.worker.min.js    # PDF.js Worker
```

### 5.2 运行方式
直接双击 `Datasheet_Comparator.html` 即可在浏览器中运行。
